# Review чеклист
1. Соответствие PEP 8 (форматирование, именование)
2. Использование type hints для функций и методов. Более 90% методов и функций должны содержать Type Hints.
3. Наименование методов и классов должно быть понятным и четко отражать суть происходящего
4. Документация (docstrings для модулей, классов, функций). В особенности для низкоуровевных методов и классов, реализующих ключевой функционал
5. Отсутствие неиспользуемого кода (dead code)
6. Правильная обработка исключений
7. Бизнес-логика должна быть покрыта тестами
