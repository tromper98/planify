# Тестирование кода
Код приложения может быть покрыт двумя видами тестов:
- Модульные тесты
- Интеграционные тесты

## Модульные тесты
Проверяют работоспособность отдельной функции/модуля/класса, для которых не требуется окружения 
или прочих зависимостей.

### Требования к тестам
1. Тесты должны покрывать основные сценарии использования
2. Негативные сценарии и пограничные случаи также должны проверятся
3. Обязательно покрываем тестами:
    - Основную логику приложения
    - Сложные и неочевидные части приложения

### Работа с модульными тестами
1. Все модульные тесты создаются в `tests/unit`
2. Каждый тест - это отдельная функция с префиксом `test_`
3. Запускать unit-тесты можно с помощью скрипта `bin/run-unit-tests`

### Работа с интеграционными тестами
1. Все интеграционные тесты создаются в `tests/integration`
2. Каждый тест - это отдельная функция с префиксом `test_`
3. Запускаются интеграционные тесты следующим образом:
    - Обновление тестового окружения `bin/pl-dev-build-test-images`
    - Запуск интеграционные тестов `bin/pl-run-integration-tests`
